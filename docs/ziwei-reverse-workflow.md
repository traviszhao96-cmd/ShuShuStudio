# 紫微命盘反推工作流

这份说明对应两个 CLI 工具：

- `npm run reverse:ziwei-time`
  适合生日已知、时辰未知。输入少量宫位星曜约束，输出候选时辰排名。
- `npm run reverse:ziwei-chart`
  适合只有命盘截图，甚至没有生日。输入截图里能确认的元信息与宫位星曜，在给定阳历区间里搜索最像的出生日期与时辰。

## 1. 生日已知时

示例：

```bash
npm run reverse:ziwei-time -- \
  --date 1996-03-19 \
  --gender male \
  --calendar solar \
  --palace "命宫:贪狼" \
  --palace "父母:天机,巨门"
```

建议：

- 先录 `命宫`、`夫妻`、`迁移`、`福德` 这些主星辨识度高的宫位。
- 若截图里辅星也看得很清楚，再补 `右弼`、`文昌`、`文曲`、`禄存` 这类信息。
- 若只想按主星比，加入 `--major-only`。

## 2. 只有截图时

示例：

```bash
npm run reverse:ziwei-chart -- \
  --from 1980-01-01 \
  --to 2010-12-31 \
  --gender male \
  --five-elements "土五局" \
  --soul 文曲 \
  --body 天梁 \
  --soul-branch 巳 \
  --major-only \
  --palace "命宫@酉:天府" \
  --palace "夫妻@未:紫微,破军" \
  --palace "子女@午:天机" \
  --palace "疾厄@辰:太阳" \
  --palace "迁移@卯:武曲,七杀" \
  --palace "交友@寅:天同,天梁" \
  --palace "官禄@丑:天相" \
  --palace "田宅@子:巨门" \
  --palace "福德@亥:廉贞,贪狼" \
  --palace "父母@戌:太阴"
```

说明：

- `@酉` 这种写法表示这个宫位在截图里的地支。
- `--five-elements` 对应中间的 `土五局 / 木三局 / 金四局 / 水二局 / 火六局`。
- `--soul` 对应 `命主`。
- `--body` 对应 `身主`。
- `--soul-branch` 目前用于录入截图中心的 `子斗:巳` 这类信息。

## 3. 录图顺序

从截图转录时，优先级建议如下：

1. 中央信息
   - `阳男 / 阴女`
   - `五行局`
   - `命主`
   - `身主`
   - `子斗`
2. 宫位地支
   - 例如 `命宫在酉`、`夫妻在未`
3. 主星
   - 例如 `紫微、破军、天府、武曲、七杀`
4. 辅星与杂曜
   - 例如 `右弼、文昌、文曲、红鸾、禄存`

如果截图清晰度一般，宁可少录，也不要把看不清的星硬塞进去。

## 4. 当前限制

- `reverse:ziwei-time` 目前主要解决“反推时辰档”，不是精确到分钟。
- `reverse:ziwei-chart` 目前是暴力搜索，大区间会比较慢。
- 只有截图、没有生日时，工具能给出“最像的候选日期/时辰”，不保证唯一真值。
- 录错一两个星曜会显著影响候选排序，所以推荐先只录高置信字段。
